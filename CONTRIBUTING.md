# Contributing

Thanks for your interest in contributing!

- Follow the existing code style.
- Run `make build` and `make test` locally before opening a PR.
- Keep changes small and focused; open issues for larger features first.
- Use topic branches named like `fix/xxx` or `feat/xxx`.

PR Process
1. Fork the repository.
2. Create a branch for your change.
3. Push your branch and open a Pull Request against `main`.
4. Describe changes, testing steps, and any potential risks.
